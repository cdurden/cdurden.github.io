Creating a Hall Pass System using Google Forms

Introduction

This document describes a hall pass system that I developed. It allows a user to login with a Google Account and request a hall pass using a Google Form. It also enables the administrator to control authorization of hall passes. As an example, in this tutorial, we limit the number per day.

I will start by describing the different components of the system. Each component, along with its function, is presented in the following table.

Component

Start a Hall Pass Google Form

The Start a Hall Pass Google Form allows users to submit a hall pass request.

The End a Hall Pass Google Form allows users to end an active hall pass.

Start a Hall Pass Google Form Responses Sheet

Records all of the hall pass requests submitted using the Start a Hall Pass Google Form.

Cryptographically Hashed Responses Sheet from the Responses Sheet

The Cryptographically Hashed Response Data Sheet contains cryptographic hashes of the hall pass data from the Responses Sheet. In addition, it contains a computed “status” field (which may take the values “active”, “inactive”, “pending”, “denied”) which are used by the system to control its behavior.

Note: This sheet is publicly-viewable by anyone with the link.

The purpose of the hashing process is to prevent users from recovering data about students who have requested hall passes.

Hall Pass Monitoring Google Sheet

This is a private Google Sheet that contains the hall pass data unencrypted, along with some computed fields (“active”, “inactive”, “duration”) regarding the hallpass states, so that authorized individuals can monitor pass information.

function toHexString(signature) {

 return signature

   .map(function(byte) {

       // Convert from 2's compliment

       var v = (byte < 0) ? 256 + byte : byte;

       // Convert byte to hexadecimal

       return ("0" + v.toString(16)).slice(-2);

   })

   .join("");

}

 var ss = SpreadsheetApp.openById("1V6tGSDFuzBwnC4z5H7HKCLONKLpOs7_hiNvT-g_Kz1c").getSheets()[0];

 var ssDigest = SpreadsheetApp.openById("1Uij5GC-HJB4wd8xMAgtxUe6QBSEkt_vsjCtuRHhok3M").getSheets()[0];

 var data = ss.getDataRange().getValues();

 var digestData = [];

 var rowDigest, row;

 for (var i = 0; i < data.length; i++) {

   row = data[i]

   if (i===0) {

     rowDigest = row;

   } else {

     rowDigest = [Utilities.formatDate(new Date(row[0]), 'America/Chicago', 'MM/dd/yyyy HH:mm:ss'), toHexString(Utilities.computeDigest(Utilities.DigestAlgorithm.SHA_256, row[1])), row[2], toHexString(Utilities.computeDigest(Utilities.DigestAlgorithm.SHA_256, row[3]))];

   }

   digestData.push(rowDigest);

 }

 ssDigest.getRange(1, 1, digestData.length, 4).setValues(digestData);

}

Conclusion

Most of the functionality of the system is situated in well-established Google web apps. However, the system also requires a web server to be configured (along with Google OAuth credentials) to serve a small amount of JavaScript code.

The system publishes encrypted student email addresses to a publicly-available Google Sheet. Before using this in a school environment, the cryptographic scheme should be augmented with at least one additional safeguard (like salting) and evaluated for security.

Also, we should verify that student email addresses are considered directory information under FERPA, such that they fall under the category of “information contained in an education record of a student that would not generally be considered harmful or an invasion of privacy if disclosed.” We should also check whether inadvertent disclosure of hall pass data would constitute a violation of FERPA. Here and here are articles about security systems to safeguard student privacy. One practice we can implement is to delete hall pass data daily.

We might also evaluate other approaches which could avoid publishing these data.
